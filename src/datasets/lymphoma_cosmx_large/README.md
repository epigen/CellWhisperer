# Lymphoma CosMx Large Dataset Pipeline

This pipeline processes 11 TMAs (8 files due to joint scanning) from the lymphoma CosMx large dataset. It follows the multi-dataset pattern similar to hest1k.

## Overview

The pipeline transforms from a single-TMA structure to a multi-TMA structure:
- **Input**: 1 combined h5ad file + 8 TIFF images (optional, one per TMA)
- **Output**: 8 separate processed h5ad files in multi-dataset format

### Optional TIFF Images

TIFF images can be made optional via the `USE_TIFF_IMAGES` flag in the Snakefile:
- Set `USE_TIFF_IMAGES = True` when images are available (enables full image processing)
- Set `USE_TIFF_IMAGES = False` to process without images (coordinate transformation only)

**Current setting**: `False` (images not yet available)

## Key Changes from lymphoma_cosmx_small

1. **Multi-TMA Processing**: Uses `{dataset}` and `{sample}` wildcards instead of single dataset
2. **Pixel-based Offsets**: Offsets are now in pixels (not mm) and applied after mm-to-pixel conversion
3. **Separate X/Y Scaling**: Uses `scale_factor_x` and `scale_factor_y` instead of single `scale_factor`
4. **Dynamic Parameters**: Alignment parameters loaded from `data/alignment.csv` per TMA
5. **Rotation**: Fixed at 270° (instead of 269.73°)

## Dataset Structure

### TMAs Included
The h5ad file contains individual TMAs (TMA1-TMA16), which are grouped for processing:

- **TMA1** (individual)
- **TMA2** (individual)
- **TMA3** (individual)
- **TMA4** = TMA4-Sec1 + TMA4-Sec2 (scanned in 2 sections)
- **TMA5** (individual)
- **TMA11_12** = TMA11 + TMA12 (joint scan)
- **TMA13_14** = TMA13 + TMA14 (joint scan)
- **TMA15_16** = TMA15 + TMA16 (joint scan)

**Total**: 8 output files representing 12 individual TMAs (TMA1-5, TMA11-16)

### Alignment Parameters

Located in `data/alignment.csv`:
- `sample`: TMA identifier
- `x_offset_pixels`: X offset in pixels
- `y_offset_pixels`: Y offset in pixels
- `scale_factor_x`: X dimension scale factor
- `scale_factor_y`: Y dimension scale factor

## Pipeline Structure

### Rule: `copy_images`
Copies TIFF images from source directory to resources.

**Input**: `/oak/stanford/groups/zinaida/CAR_T_data_warehouse/CosMX/stanford_CART_TMA/CosMx_CAR_T_post_run_HE/exported_tiff_wsi/{sample}.tiff`

**Output**: `resources/lymphoma_cosmx_large/images/{sample}.tiff`

### Rule: `disentangle_tmas`
Splits the combined h5ad file into per-TMA h5ad files, grouping TMAs according to alignment.csv.

This file was generated by Eric through integration across all TMAs for batch effect removal using scTransform.

**Input**: `resources/lymphoma_cosmx_large/adata_combined_seurat.h5ad`

**Output**: `resources/lymphoma_cosmx_large/per_tma_h5ads/{sample}.h5ad` (8 files)

**Parameters**:
- `tma_column`: Column name in `adata.obs` identifying individual TMA (default: "tmaID")
- `tma_grouping`: Dictionary mapping output sample names to lists of individual TMA IDs:
  - `TMA4` ← `["TMA4-Sec1", "TMA4-Sec2"]`
  - `TMA11_12` ← `["TMA11", "TMA12"]`
  - `TMA13_14` ← `["TMA13", "TMA14"]`
  - `TMA15_16` ← `["TMA15", "TMA16"]`
  - Others remain individual (TMA1, TMA2, TMA3, TMA5)

### Rule: `process_data`
Processes individual TMA data with per-TMA alignment parameters.

**Input**:
- `resources/lymphoma_cosmx_large/per_tma_h5ads/{sample}.h5ad`
- `resources/lymphoma_cosmx_large/images/{sample}.tiff`

**Output**:
- `results/lymphoma_cosmx_large/h5ads/full_data_{sample}.h5ad`
- `results/lymphoma_cosmx_large/images/{sample}.tiff`
- QC plots and patches

**Parameters** (per TMA from alignment.csv):
- `x_offset_pixels`, `y_offset_pixels`
- `scale_factor_x`, `scale_factor_y`
- Plus standard parameters: `spot_diameter_pixels`, `white_cutoff`, `pixel_size_um`, etc.

## Usage

### Prerequisites

1. Place the combined h5ad file at:
   ```
   resources/lymphoma_cosmx_large/adata_combined_seurat.h5ad
   ```
   (This file was generated by Eric through integration across all TMAs for batch effect removal using scTransform)

2. **(Optional)** Set `USE_TIFF_IMAGES = True` in Snakefile when TIFF images are available at:
   ```
   /oak/stanford/groups/zinaida/CAR_T_data_warehouse/CosMX/stanford_CART_TMA/CosMx_CAR_T_post_run_HE/exported_tiff_wsi/
   ```

3. Verify the TMA identifier column name in the h5ad file matches the `tma_column` parameter in the `disentangle_tmas` rule (default: "tmaID")

### Configuration

**Toggle TIFF Image Processing:**

In `Snakefile`, line 19:
```python
USE_TIFF_IMAGES = False  # Set to True when images are ready
```

When `USE_TIFF_IMAGES = False`:
- Coordinate transformation still works (converts mm to pixels)
- No background filtering (uses all grid coordinates)
- Placeholder images created for outputs
- QC plots show placeholders

When `USE_TIFF_IMAGES = True`:
- Full image processing enabled
- Background filtering applied
- Real patch extraction
- Comprehensive QC visualizations

### Running the Pipeline

```bash
cd src/datasets/lymphoma_cosmx_large

# Dry run to check everything
pixi run snakemake -n

# Run the full pipeline
pixi run snakemake --cores 8

# Run specific rule
pixi run snakemake --cores 8 disentangle_tmas
pixi run snakemake --cores 8 results/lymphoma_cosmx_large/h5ads/full_data_TMA1.h5ad
```

## Modified Files

### Created Files
1. `scripts/disentangle_tmas.py` - Splits combined h5ad into per-TMA files

### Modified Files
1. `Snakefile` - Complete rewrite for multi-TMA processing
2. `scripts/coordinate_utils.py` - Updated `convert_mm_to_pixel_coordinates()` for:
   - Pixel-based offsets (not mm-based)
   - Separate X/Y scale factors
   - 270° rotation
3. `scripts/process_data.py` - Updated to use new parameter names
4. `config.yaml` - Added `lymphoma_cosmx_large` to dataset mappings

### Coordinate Transformation Changes

**Old (lymphoma_cosmx_small)**:
```python
# Step 1: Apply mm offsets
x_coords_offset = x_mm + x_offset
y_coords_offset = y_mm + y_offset

# Step 2: Convert to pixels (proportional scaling)
mm_to_pixel = (image_width / 10.0) * scale_factor
x_coords_px = x_coords_offset * mm_to_pixel
y_coords_px = y_coords_offset * mm_to_pixel
```

**New (lymphoma_cosmx_large)**:
```python
# Step 1: Convert mm to pixels (independent X/Y scaling)
mm_to_pixel_x = (image_width / 10.0) * scale_factor_x
mm_to_pixel_y = (image_height / 10.0) * scale_factor_y
x_coords_px = x_mm * mm_to_pixel_x
y_coords_px = y_mm * mm_to_pixel_y

# Step 2: Apply pixel-based offsets
x_coords_px += x_offset_pixels
y_coords_px += y_offset_pixels
```

## Troubleshooting

### Missing Input File Error
If you see `MissingInputException for rule disentangle_tmas`, ensure the combined h5ad file exists at:
```
resources/lymphoma_cosmx_large/adata_combined_seurat.h5ad
```

### TMA Column Not Found
If the disentangle script fails with "Column not found", check the actual column name in your h5ad file and update the `tma_column` parameter in the `disentangle_tmas` rule.

### Missing TIFF Images
Ensure all 8 TIFF files exist in the source directory or the `copy_images` rule will fail. Check the source path:
```
/oak/stanford/groups/zinaida/CAR_T_data_warehouse/CosMX/stanford_CART_TMA/CosMx_CAR_T_post_run_HE/exported_tiff_wsi/
```

## Configuration

The pipeline uses the `detailed_resolution` configuration from `config.yaml`:
- `target_pixel_size`: 0.25 μm/pixel (40x magnification scaled to 20x equivalent)
- `patch_size_pixels`: 224

This matches the settings used for `lymphoma_cosmx_small_detailed`.
